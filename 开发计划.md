## 阶段 4: 登录页 (LoginView)

### 目标
实现 H5 CAS 登录流程

### 原逻辑梳理（参考 SCPlatform/LoginH5VC.m）

**登录流程**：
1. 加载 CAS 认证页面：`https://ids-hfit-edu-cn-s.hfit.edu.cn/authserver/mobile/auth?appId={appId}`
2. 用户输入账号密码后，CAS 返回包含 `mobile_code` 的 URL
3. 截获 `mobile_code`，调用 `getTokenByCode` API 换取 token
4. 保存 token，调用 `userProfile` API 获取用户信息
5. 保存 Cookie 供 H5 使用

**关键实现**：
- WKNavigationDelegate 监听 URL 变化，检测 `mobile_code=`
- 提取 code 后调用 API: `/zhxy-information/mobile/getTokenByCode`
- 获取用户信息: `/zhxy-information/mobile/userProfile`

## 布局原则（全局适用）

✅ **使用 SwiftUI 自适应布局**
- VStack/HStack/LazyVGrid/ScrollView
- .padding() / .padding(.horizontal, 16)
- AsyncImage + .resizable().aspectRatio()
- .font(.title) / .font(.body) 语义化字体
- Color(.systemBackground) 系统色适配深色模式

❌ **禁止使用**
- 固定屏幕宽度计算（如 autoWidth(375)）
- 手动计算 statusBarHeight
- 固定像素值（除非必要，如 Banner 高度）
