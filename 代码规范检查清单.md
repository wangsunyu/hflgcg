# 代码规范检查清单

每个阶段开发完成后，用 `/simplify` 审查前先自查本清单，避免反复犯同类问题。

---

## 一、Swift 并发（Concurrency）

### ✅ @MainActor 类内部无需再次调度

```swift
// ❌ 错误：类已是 @MainActor，Task 内 await MainActor.run 是多余的
@MainActor class MyViewModel: ObservableObject {
    func load() {
        Task {
            let result = try await api.fetch()
            await MainActor.run { self.data = result }  // 冗余
        }
    }
}

// ✅ 正确：直接赋值，Swift 并发保证已在主线程
@MainActor class MyViewModel: ObservableObject {
    func load() {
        Task {
            let result = try await api.fetch()
            self.data = result  // 已在 @MainActor 上下文
        }
    }
}
```

### ✅ Timer 回调用 MainActor.assumeIsolated

`Timer.scheduledTimer` 在主 RunLoop 触发，实际运行在主线程，但 Swift 编译器不知道这一点。

```swift
// ❌ 错误：编译警告，且多余的 Task 开销
countdownTimer = Timer.scheduledTimer(withTimeInterval: 1.0, repeats: true) { [weak self] _ in
    Task { @MainActor [weak self] in self?.tick() }
}

// ✅ 正确：明确告知编译器已在 MainActor 上
countdownTimer = Timer.scheduledTimer(withTimeInterval: 1.0, repeats: true) { [weak self] _ in
    MainActor.assumeIsolated { self?.tick() }
}
```

## 二、@Published 状态管理

### ✅ 只暴露 View 实际用到的状态

```swift
// ❌ 错误：isLoading、loadFailed 从未在 View 中读取
@Published var isLoading: Bool = true
@Published var loadFailed: Bool = false

// ✅ 正确：删除未使用的 @Published，保持 ViewModel 精简
```

### ✅ 可派生的状态不重复存储

```swift
// ❌ 错误：showAd 可从 adImageURL 派生，不需要单独存储
@Published var adImageURL: String?
@Published var showAd: Bool = false

// ✅ 正确：在 View 层用 if let 判断
if let imageURL = viewModel.adImageURL {
    adImageView(url: imageURL)
    skipButton
}
```

> 例外：如果 showAd 的生命周期与 adImageURL 不完全一致（如动画需要），可保留，但需注释说明原因。

## 三、持久化（UserDefaults）

### ✅ 缓存读写必须对称

```swift
// ❌ 错误：写入时没有存 cachedImageData，读取时却检查它，缓存永远失效
func cacheAdData(imageURL: String) {
    UserDefaults.standard.set(imageURL, forKey: Keys.cachedImageURL)
    // 没有写 Keys.cachedImageData
}

func loadCache() {
    if let url = UserDefaults.standard.string(forKey: Keys.cachedImageURL),
       let _ = UserDefaults.standard.data(forKey: Keys.cachedImageData) {  // 永远 nil！
        ...
    }
}

// ✅ 正确：只检查实际写入过的 key
func loadCache() {
    if let url = UserDefaults.standard.string(forKey: Keys.cachedImageURL) {
        ...
    }
}
```

### ✅ Keys 枚举只定义实际使用的 key

Keys 枚举中每个常量必须对应至少一处写入和一处读取，否则删除。

## 四、命名规范

### ✅ 标识符全部使用英文

```swift
// ❌ 错误：混合中文标识符，无法被 Xcode 重构工具处理，影响搜索
@Published var ad跳转链接: String?
static let cached跳转链接 = "cachedAd跳转链接"

// ✅ 正确：使用英文，注释说明业务含义
@Published var adJumpURL: String?       // 广告跳转链接
static let cachedJumpURL = "cachedAdJumpURL"
```

## 五、布局（SwiftUI）

### ✅ 禁止用 UIScreen.main.bounds 做布局计算

`UIScreen.main` 在 iOS 16+ 被标记为 deprecated，且在计算属性中每次 View body 求值都会调用。

```swift
// ❌ 错误：违反项目规范，每次渲染都调用
private var launchImageName: String {
    let h = UIScreen.main.bounds.height  // deprecated + 每次计算
    ...
}

// ✅ 正确：通过 GeometryReader 的 size 参数传入，由 SwiftUI 管理
Image(launchImageName(for: geometry.size))

private func launchImageName(for size: CGSize) -> String {
    switch size.height { ... }
}
```

### ✅ 合并同一条件的多个 if 块

```swift
// ❌ 错误：两次判断 showAd，多一次条件求值
if viewModel.showAd, let imageURL = viewModel.adImageURL {
    adImageView(url: imageURL)
}
if viewModel.showAd {
    skipButton
}

// ✅ 正确：合并到同一个 if let 块
if let imageURL = viewModel.adImageURL {
    adImageView(url: imageURL)
    skipButton
}
```

### ✅ 使用 AppTheme 常量，不硬编码数值

```swift
// ❌ 错误
.font(.system(size: 16, weight: .medium))
.padding(.trailing, 20)
.padding(.bottom, 40)

// ✅ 正确
.font(.system(size: AppTheme.fontSizeLarge, weight: .medium))
.padding(.trailing, AppTheme.paddingMedium)
.padding(.bottom, AppTheme.paddingXLarge)
```

AppTheme 现有常量参考：

| 常量 | 值 | 用途 |
|------|-----|------|
| `paddingSmall` | 8 | 紧凑间距 |
| `paddingMedium` | 16 | 标准间距 |
| `paddingLarge` | 24 | 宽松间距 |
| `paddingXLarge` | 32 | 超大间距 |
| `fontSizeSmall` | 12 | 辅助文字 |
| `fontSizeMedium` | 14 | 正文 |
| `fontSizeLarge` | 16 | 强调文字 |
| `fontSizeXLarge` | 18 | 小标题 |
| `fontSizeTitle` | 20 | 标题 |
| `cornerRadiusSmall` | 8 | 小圆角 |
| `cornerRadiusMedium` | 12 | 标准圆角 |
| `cornerRadiusLarge` | 16 | 大圆角 |

## 六、代码组织

### ✅ Combine 只在实际用到时才 import

```swift
// ❌ 错误：import Combine 但没有使用任何 Combine API
import Combine
private var cancellables = Set<AnyCancellable>()  // 从未被使用

// ✅ 正确：@MainActor + Task 已足够，不需要 Combine
```

> 注意：CLAUDE.md 规范要求「所有 ObservableObject 类需要 import Combine」——但如果完全不用 Combine 操作符和 sink，仅靠 @Published + Task 的类可以不 import。有实际 Combine 使用时才引入。

### ✅ 数据模型放在 Core/Models/，不放在 ViewModel 文件内

```
// ❌ 错误：Advertisement、AdData 定义在 SplashViewModel.swift 内
// ✅ 正确：提取到 Core/Models/AdvertisementModel.swift
```

## 七、检查流程（每阶段完成后）

```
□ 所有 @Published 属性都有对应的 View 读取点
□ UserDefaults 的每个 key 都有对称的写入和读取
□ @MainActor 类内部无多余的 await MainActor.run
□ Timer 回调使用 MainActor.assumeIsolated
□ 无中文标识符（变量名、常量名、枚举 case）
□ 无 UIScreen.main.bounds 布局计算
□ 数值 padding/fontSize 使用 AppTheme 常量
□ 数据模型定义在 Core/Models/
□ 编译无 error，无 warning（appintentsmetadataprocessor 除外）
```
